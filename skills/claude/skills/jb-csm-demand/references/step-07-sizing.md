# Step 7: 规模评估

## 策略

经贝采用 **Story Points** 作为需求规模评估方法，评估序列为 **1/2/3/5/8**（斐波那契简化版）。基于收集到的信息，自动评估规模。**不需要询问用户，直接根据内容判断。**

## 评估基准

| Points | 工作量定义 | 典型特征 |
|--------|-----------|----------|
| **1** | 单表增删改查，简单字段调整 | 只涉及一个数据表，前端1个简单页面或字段调整，无复杂逻辑 |
| **2** | 涉及2-3个表，有简单业务逻辑 | 需要多表关联查询，有简单的条件判断或计算逻辑 |
| **3** | 复杂业务逻辑，需新增接口和页面 | 需要设计新接口、新页面，涉及状态流转或权限控制 |
| **5** | 跨模块改动，需数据迁移或接口联调 | 涉及多个模块配合，或需要数据迁移，或对接第三方系统 |
| **8** | 大型功能，需技术方案设计 | 涉及多个系统联动，需要先做技术调研或方案设计，周期超过1周 |

## 评估规则

根据需求描述中的关键词和复杂度判断：

```
IF 需求是 "加一个字段" / "改个文案" / "调整显示格式" → 1 point
IF 需求涉及 "新增一张表的数据维护" → 1 point
IF 需求涉及 "两个表关联查询展示" / "简单筛选" → 2 points
IF 需求涉及 "新增一个功能模块" / "审批流程调整" / "报表导出" → 3 points
IF 需求涉及 "跨系统对接" / "数据迁移" / "批量处理大量数据" → 5 points
IF 需求涉及 "重构" / "技术方案待确定" / "多个模块联动改造" → 8 points
```

## 输出格式

> **规模评估：X Points**
>
> 评估依据：[简要说明为什么是这个分数，参考了哪些描述]
