# Step 9: 信息完整性检查（强制）

## 策略

**在生成最终文档前，必须对信息传递的完整性进行自动评估。** 此步骤由 AI 自动完成，不需要 CSM 具备产品专业知识——CSM 只需根据提示补充缺失信息。

## 评估逻辑

AI 根据已收集的内容，自动检查以下 5 个维度，每项 20 分，满分 100 分：

| 维度 | 检查内容 | 满分标准 |
|------|---------|---------|
| **客户信息** | 名称、级别、机构ID、行业、反馈人角色 | 5 项齐全 |
| **场景描述** | 角色、情境、操作三要素 | 3 要素齐全且具体 |
| **痛点描述** | 问题、影响、workaround | 问题+影响明确，workaround 可不填 |
| **期望方案** | 期望效果、竞品参考 | 期望效果明确，竞品参考可不填 |
| **补充信息** | 阻断性、紧急程度、影响客户数 | 3 项齐全 |

## 评分规则

```
客户信息（20分）：
  - 5 项齐全 → 20分
  - 缺 1 项 → 16分
  - 缺 2 项 → 12分
  - 缺 3 项 → 8分
  - 缺 4 项及以上 → 4分

场景描述（20分）：
  - 三要素齐全且具体 → 20分
  - 三要素齐全但某要素模糊 → 15分
  - 缺 1 个要素 → 10分
  - 缺 2 个及以上 → 5分

痛点描述（20分）：
  - 问题+影响均明确 → 20分
  - 问题明确，影响模糊 → 15分
  - 仅有问题，无影响 → 10分
  - 问题和影响均缺失 → 5分

期望方案（20分）：
  - 期望效果明确 → 20分
  - 期望效果模糊但方向清楚 → 15分
  - 仅有简单提及，无具体方向 → 10分
  - 完全未提及期望 → 5分

补充信息（20分）：
  - 3 项齐全 → 20分
  - 缺 1 项 → 15分
  - 缺 2 项 → 10分
  - 缺 3 项 → 5分
```

## 输出格式

直接展示检查结果，**不需要 CSM 打分**：

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

客户信息：    XX/20 分  [✓ 完整 / ⚠ 缺X项]
场景描述：    XX/20 分  [✓ 完整 / ⚠ 缺X项]
痛点描述：    XX/20 分  [✓ 完整 / ⚠ 缺X项]
期望方案：    XX/20 分  [✓ 完整 / ⚠ 缺X项]
```

## 结果处理

**总分 ≥ 70 分（达标）：**
> ✅ 信息传递完整度 {XX}/100%，达到提交标准。
> 将直接生成需求文档。

**总分 60 ~ 69 分（合格）：**
> ⚠️ 信息传递完整度 {XX}/100%，基本合格，但存在以下缺口：
> - {具体列出缺失项}
>
> 你可以选择：
> 1. **现在补充** → 告诉我缺失的信息，更新后重新检查
> 2. **直接生成** → 文档中会标注 "信息待补充"，产品部可能退回要求补充

**总分 < 60 分（不合格，不允许提交）：**
> ❌ 信息传递完整度 {XX}/100%，**未达到提交标准**。
>
> 关键缺失：
> - {具体列出缺失项}
>
> 请补充以上信息后再生成文档。如需重新开始，输入 `/jb-csm-demand`。
>
> **⚠️ 注意：低于 60 分的需求不允许提交**，必须补充信息至 60 分以上才能生成文档。

## 强制阻断规则

- 总分 **< 60 分**：**禁止生成文档**，不存在任何绕过方式。CSM 必须补充信息使总分达到 60 分以上。
- 场景描述 **三要素缺失 2 个及以上**：即使总分达标，也强制要求补充场景信息。
