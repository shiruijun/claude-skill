# Step 5.9: 汇总确认（合并确认环节）

## 策略

将 Step 5.5（用户故事）、Step 5.7（功能模块）、Step 6（补充信息）合并为**一次汇总确认**，减少 CSM 确认次数。

## 执行时机

在 Step 5（期望方案）收集完成后，AI 自动：
1. 生成用户故事（Step 5.5）
2. 匹配功能模块（Step 5.7）
3. 准备补充信息问题（Step 6）

然后一次性展示汇总信息，让 CSM 确认或修改。

## 汇总展示格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 信息汇总确认（请检查以下信息）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【用户故事】（AI 自动生成）
As a {角色}，
I want {期望效果提炼}，
So that {痛点解决后的价值}。

【功能模块】（AI 自动匹配）
一级模块：{猜测结果}
二级模块：{猜测结果}

【补充信息】
1. 阻断性：是/否
2. 紧急程度：紧急/高/中/低
3. 影响客户数：仅1家 / 2-5家 / 6-20家 / 20+家
4. 竞品已支持：是/否/不清楚
5. 影响续费：是/否

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 确认交互

> 以上信息由 AI 根据你提供的内容自动生成，请确认：
>
> 1. **全部准确** → 直接生成需求文档
> 2. **需要修改** → 告诉我哪项不对，格式如「修改用户故事：xxx」或「修改紧急程度：高」
> 3. **补充客户原话** → 直接发送原话内容
> 4. **上传附件** → 直接发送图片或文件

## 修改规则

CSM 可以直接发送修改指令，支持以下格式：

| 修改指令 | 示例 |
|----------|------|
| 修改用户故事：xxx | 修改用户故事：As a 采购经理... |
| 修改一级模块：xxx | 修改一级模块：财务中心 |
| 修改二级模块：xxx | 修改二级模块：付款管理 |
| 修改阻断性：是/否 | 修改阻断性：是 |
| 修改紧急程度：xxx | 修改紧急程度：紧急 |
| 修改影响范围：xxx | 修改影响范围：2-5家 |
| 修改竞品：xxx | 修改竞品：用友已支持 |
| 修改续费影响：是/否 | 修改续费影响：是 |

AI 收到修改指令后更新对应字段，并重新展示汇总信息供确认。
