# Step 2: 收集客户信息

## 策略

**优先检测批量输入：** 收到用户消息后，先判断是否包含模板格式的完整信息。如果是，自动提取所有字段，跳过逐问流程。

**批量输入识别信号（满足以下任一条件即判定为批量模式）：**
1. 包含模板格式的字段标记（如 `**客户名称**：`、`**使用场景**：` 等）
2. 同时包含「客户名称」+「使用场景/场景」+「痛点/期望」3 个关键词
3. 包含 5 个及以上提取字段（见下方表格）

**批量输入处理流程：**
```
用户输入 → 提取所有字段 → 信息完整度预检 → 跳过后续逐问步骤
```

---

## 批量模式（自动提取）

当检测到用户一次性粘贴了完整信息时，按以下规则提取字段：

| 提取字段 | 匹配关键词 | 示例 |
|----------|-----------|------|
| 客户名称 | `客户名称`、`客户：`、`客户公司` | `客户名称：测试科技` |
| 客户级别 | `客户级别`、`客户等级`、`级别：` | `客户级别：战略` |
| 客户机构ID | `客户ID`、`机构ID`、`record_id` | `机构ID：12345` |
| 行业 | `行业`、`所属行业` | `行业：制造业` |
| 反馈人角色 | `角色`、`反馈人`、`联系人` | `角色：财务总监` |
| 使用场景 | `使用场景`、`场景`、`谁在什么情况下` | `场景：财务月底对账时` |
| 当前痛点 | `痛点`、`当前痛点`、`遇到什么问题` | `痛点：只能逐条导出` |
| 期望方案 | `期望`、`期望方案`、`希望系统` | `期望：支持批量导出` |
| 影响范围 | `影响范围`、`影响客户数` | `影响范围：多家客户` |
| 紧急程度 | `紧急程度`、`优先级`、`紧急` | `紧急程度：高` |
| 客户原话 | `客户原话`、`原话`、`客户说` | `原话："我们每月要花3天..."` |
| 附件说明 | `附件`、`截图`、`文件` | `附件：见截图` |

**提取后处理：**
- 所有字段提取完成后，向 CSM 确认提取结果（列出关键信息，不逐项确认）
- 直接进入 Step 2.6 客户名称关联
- 批量模式下不再执行逐问流程

---

## 逐问模式（兜底）

未检测到批量输入时，按顺序询问：

| 序号 | 问题 | 说明 |
|------|------|------|
| 2.1 | **客户名称**是什么？ | 记录客户公司全称或简称 |
| 2.2 | **客户级别**？ | 战略 / 价值 / 商业 / 其他（请说明） |
| 2.3 | **客户机构ID**？ | 客户在系统中的唯一标识 |
| 2.4 | 所属**行业**？ | 便于归类分析 |
| 2.5 | **反馈人角色**？ | 如：财务总监、采购经理、CEO |

## Step 2.6: 客户名称关联确认（自动）

收到客户名称后，自动搜索飞书客户表（`tbl698sXiRBfZuqg`）进行匹配：

```bash
lark-cli base +record-search \
  --as user \
  --base-token IFusbLuzuaCLkRsMs4NcgoEznYg \
  --table-id tbl698sXiRBfZuqg \
  --query "<客户名称>"
```

### 搜索结果处理

- **匹配到 1 条且名称高度相似** → 自动关联，向 CSM 确认：
  > 已找到客户「xxx」，确认是这个客户吗？（是/否）

- **匹配到多条** → 列出候选（最多 5 条），让 CSM 选择：
  > 找到多个匹配客户，请选择：
  > 1. xxx
  > 2. xxx
  > 3. 以上都不是

- **未匹配到** → 记录该客户名称，后续写入 `来源备注` 字段：
  > 未在客户表中找到「xxx」，已记录到备注中，产品部可后续补充关联。

**确认结果：** 记录目标客户的 `record_id`（`fldWFlVBDa` link 字段值）。如果未匹配到，标记 `customer_link_id = null`。
