# Step 8: RICE 评分

## 策略

基于 Step 6 的补充信息，自动计算 RICE Score。**不需要询问用户，直接根据已有信息判断。**

## 评分标准

### R - Reach（触达范围）

| 分值 | 定义 | 判定条件 |
|------|------|----------|
| 0.5 | 仅1个客户 | 6.3 = "仅1家" |
| 1 | 2-5个客户 | 6.3 = "2-5家" |
| 2 | 6-20个客户 | 6.3 = "6-20家" |
| 3 | 20+个客户 | 6.3 = "20+家" |

### I - Impact（影响程度）

| 分值 | 定义 | 判定条件 |
|------|------|----------|
| 0.25 | 边缘改善 | 非阻断性 + 低紧急 + 不影响续费 |
| 0.5 | 小幅优化 | 非阻断性 + 中紧急 |
| 1 | 中等改善 | 非阻断性 + 高紧急；或 阻断性 + 低紧急 |
| 2 | 显著提升 | 阻断性 + 中紧急；或非阻断 + 高紧急 + 影响续费 |
| 3 | 重大/阻断性 | 阻断性 +（高/紧急）紧急；或明确影响续费/增购 |

> **判定优先级：** 影响续费 > 阻断性 > 紧急程度

### C - Confidence（信心度）

| 分值 | 定义 | 判定条件 |
|------|------|----------|
| 50% | 需求较模糊 | 场景/痛点/期望任一要素不清晰，或客户描述含糊 |
| 80% | 需求基本明确 | 三要素基本清晰，但缺少具体方案细节或数据支撑 |
| 100% | 需求非常明确 | 三要素清晰，有具体操作路径，或有多客户反馈验证 |

### E - Effort（工作量）

直接使用 Step 7 评估的 Story Points 数值。

## 计算公式

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

## 输出格式

```markdown
### RICE 评分

| 维度 | 分值 | 说明 |
|------|------|------|
| Reach（触达） | X | 影响 Y 个客户 |
| Impact（影响） | X | [阻断性/紧急程度/续费影响说明] |
| Confidence（信心） | X% | [需求明确程度说明] |
| Effort（工作量） | X | X Story Points |

**RICE Score = (X × X × X%) / X = X.X**
```

## 优先级解读

| RICE Score | 优先级 | 建议 |
|-----------|--------|------|
| ≥ 3.0 | P0 - 最高 | 建议尽快排入迭代 |
| 1.5 ~ 3.0 | P1 - 高 | 建议下迭代排期 |
| 0.5 ~ 1.5 | P2 - 中 | 放入需求池，按资源情况排期 |
| < 0.5 | P3 - 低 | 记录归档，等待更多客户反馈 |
