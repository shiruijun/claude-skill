# 安全指南

## AI Coding 安全红线（最优先）

在使用 AI 辅助编程时，以下安全红线**绝对禁止**：

1. **严禁**将数据库连接密码、API 密钥、内部接口地址作为上下文发送给任何外部 AI 服务
2. **严禁**在代码中硬编码密码或密钥，必须使用环境变量或配置中心的占位符（如 `${xxx}`）
3. **严禁**将包含敏感信息的配置文件（`bootstrap.yml`、`application.yml`、`.env` 等）提交到 Git
4. 敏感信息（Token、密码、手机号、身份证号、银行卡号）禁止打印到日志或输出到 console
5. 所有用户输入必须经过校验后再处理，禁止直接拼接 SQL
6. 文件上传接口必须校验文件类型和大小，禁止信任客户端提供的文件名后缀

## 强制安全检查

在任何提交之前：
- [ ] 无硬编码密钥（API 密钥、密码、令牌）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（净化 HTML）
- [ ] CSRF 保护已启用
- [ ] 认证/授权已验证
- [ ] 所有端点启用速率限制
- [ ] 错误消息不泄露敏感数据

## 密钥管理

- 永远不要在源代码中硬编码密钥
- 始终使用环境变量或密钥管理器
- 启动时验证所需的密钥是否存在
- 轮换任何可能已暴露的密钥

## 安全响应协议

如果发现安全问题：
1. 立即停止
2. 使用 **security-reviewer** agent
3. 在继续之前修复关键问题
4. 轮换任何已暴露的密钥
5. 审查整个代码库中的类似问题
