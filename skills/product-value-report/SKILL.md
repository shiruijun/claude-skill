# 产品业务价值报告生成器

## Overview

将需求文档转化为标准化的产品业务价值报告，采用"业务现状 → 产品目标 → 本次方案 → 业务价值"四段式结构，业务价值从降本、增效、控险、增收四个维度分析。

## When to Use

- 用户提供需求文档（PRD/设计文档）并要求提炼业务价值
- 用户输入 `/业务价值` 或类似指令
- 需要将技术方案转化为业务语言向利益相关者沟通

## 核心框架

```
业务现状 → 产品目标 → 本次方案 → 业务价值（降本/增效/控险/增收）
```

## Output Format

```markdown
## 业务现状

[描述当前痛点或问题场景，2-3个典型场景]

---

## 产品目标

[一句话描述本次产品的核心目标]

---

## 本次方案

[简要描述实现方式，3-5个关键特性/机制]

---

## 业务价值

| 维度 | 业务价值 |
|------|----------|
| **降本** | [具体节省了什么，减少了什么成本] |
| **增效** | [具体提升了什么效率，简化了什么流程] |
| **控险** | [具体控制了哪些风险，保障了什么合规] |
| **增收** | [间接如何支撑业务增长或收入提升] |

---

**下载路径**：`[自动生成的报告文件路径]`
```

## 执行步骤

### Step 1: 读取需求文档

读取用户提供的需求文档完整内容。

### Step 2: 提炼业务价值报告

按照以下顺序分析并生成报告：

1. **业务现状**：从"背景"、"问题"章节提取痛点场景（2-3个），区分问题类型（效率问题/质量风险/能力缺失），量化痛点
2. **产品目标**：从"产品目标"章节提取核心目标，用一句话概括（动词+动作+期望结果）
3. **本次方案**：从"功能范围"、"详细设计"章节提取3-5个关键特性，用动宾短语描述
4. **业务价值**：从降本、增效、控险、增收四个维度分析，每个维度给出具体可落地的价值描述

### Step 3: 生成报告并保存

1. 将报告渲染为 Markdown 格式
2. **确定输出路径**：取需求文档所在目录，文件名为 `{原文件名}-业务价值报告.md`
3. 将报告写入输出路径
4. 在报告末尾附加下载路径信息

## 业务现状提炼方法

1. **找痛点场景**：从文档中提取实际业务痛点（通常在"背景"、"问题"章节）
2. **区分问题类型**：
   - 效率问题：操作繁琐、重复劳动、等待时间长
   - 质量风险：数据错误、统计口径不一致、流程漏洞
   - 能力缺失：现有功能无法支持新业务场景
3. **量化痛点**：如"每月手动填写几十行"→ 说明工作量

## 产品目标提炼方法

1. 从文档的"产品目标"或"需求概述"中提取核心目标
2. 一句话概括：动词 + 动作 + 期望结果
3. 示例："在高级分摊功能中新增业务科目作为分摊对象类型，支持事后分摊场景"

## 方案提炼方法

1. 提取关键新功能/特性（通常在"功能范围"、"详细设计"章节）
2. 列出3-5个核心机制或特性
3. 用简洁的动宾短语描述：XX功能、支持XX、实现XX

## 业务价值分析维度

| 维度 | 分析角度 | 关键词 |
|------|----------|--------|
| **降本** | 减少人力、时间、物资成本 | 节省、减少、避免、自动化 |
| **增效** | 提升效率、缩短周期、简化操作 | 简化、一键、自动化、缩短、提升 |
| **控险** | 控制风险、保障合规、数据准确 | 校验、核实、保障、确保、统一 |
| **增收** | 间接支撑业务增长 | 支撑、赋能、释放、专注、高价值 |

## Quick Reference

**分析顺序**：业务现状(找痛点) → 产品目标(一句话) → 本次方案(列特性) → 业务价值(填四维)

**价值撰写原则**：
- 降本/增效：聚焦直接可测量的改善
- 控险/增收：聚焦长期价值和战略意义
- 每条价值需具体、可落地，避免空洞描述

**输出文件名规则**：`{原需求文档名}-业务价值报告.md`
